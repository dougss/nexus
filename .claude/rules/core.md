# Core

Leia `CLAUDE.md` na raiz do projeto antes de qualquer ação. Se não existir, rode `/setup` antes.

Respeite a organização existente. Nunca crie pastas ou arquivos fora da estrutura já estabelecida sem aprovação. Siga o padrão de naming e localização dos arquivos similares existentes.

Arquivos: kebab-case. Variáveis/funções: camelCase. Tipos: PascalCase. Colunas DB: snake_case.

Mantenha imports organizados. No server, sempre use extensão `.js` nos imports mesmo para arquivos `.ts` — requisito ESM Node. Não altere código fora do escopo da tarefa.

Código: inglês. Commits: seguir conventional commits (`feat`, `fix`, `refactor`, `docs`, `test`, `chore`). Comunicação: português (BR).

Se existir plano ativo em `.claude/plans/_active.md`, seguir os steps definidos.
