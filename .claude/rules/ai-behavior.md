# Comportamento

Antes de qualquer mudança: confirmar entendimento em 2-3 frases, identificar arquivos afetados. Para tarefas complexas (3+ arquivos, mudança de arquitetura): gerar plano em `.claude/plans/` antes de executar.

Se a tarefa tem mais de uma interpretação, perguntar. Se detectar risco ou trade-off, comunicar antes de prosseguir. Não assumir requisitos não declarados.

Começar pelo diagnóstico: ler logs, reproduzir o erro, entender o fluxo antes de propor solução. Preferir a solução mais simples. Não adicionar complexidade especulativa.

Ler arquivos relevantes antes de sugerir mudanças. Verificar referências. Nunca modificar fora do escopo sem avisar.

Não inventar APIs ou funcionalidades. Se não tem certeza, verificar docs ou código existente. "Não sei" > código plausível mas errado.
