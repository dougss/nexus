**Goal:** Replace the Nexus MCP server and Superpowers plugin with a single native Claude Code plugin using Markdown skills, SessionStart hook, and enforcement patterns.

**Tech Stack:** Bash (hook), Node.js 22 (CLI, zero external deps), Markdown (skills)

- [ ] **Step 2: Create session-start script**

Create `hooks/session-start`:

```bash
#!/bin/bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")/.." && pwd)"
SKILL_FILE="$SCRIPT_DIR/skills/using-nexus/SKILL.md"

if [ ! -f "$SKILL_FILE" ]; then
  echo '{}' >&2
  exit 0
fi

CONTENT=$(cat "$SKILL_FILE")

# Escape for JSON: backslashes, quotes, newlines, tabs
ESCAPED=$(printf '%s' "$CONTENT" | sed 's/\\/\\\\/g; s/"/\\"/g; s/	/\\t/g' | awk '{printf "%s\\n", $0}' | sed '$ s/\\n$//')

cat <<HOOK_JSON
{
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "<EXTREMELY_IMPORTANT>\nYou have Nexus skills available.\n\n**Below is the full content of your 'nexus:using-nexus' skill — your introduction to using skills. For all other skills, use the 'Skill' tool:**\n\n---\n${ESCAPED}\n\n</EXTREMELY_IMPORTANT>"
  }
}
HOOK_JSON
```

## Instruction Priority

Nexus skills override default system prompt behavior, but **user instructions always take precedence**:

1. **User's explicit instructions** (CLAUDE.md, direct requests) — highest priority
2. **Nexus skills** — override default system behavior where they conflict
3. **Default system prompt** — lowest priority

## How to Access Skills

Use the `Skill` tool. When you invoke a skill, its content is loaded and presented to you — follow it directly. Never use the Read tool on skill files.

## The Rule

**Invoke relevant or requested skills BEFORE any response or action.** Even a 1% chance a skill might apply means that you should invoke the skill to check. If an invoked skill turns out to be wrong for the situation, you don't need to use it.

## Red Flags

These thoughts mean STOP — you're rationalizing:

| Thought                             | Reality                                                |
| ----------------------------------- | ------------------------------------------------------ |
| "This is just a simple question"    | Questions are tasks. Check for skills.                 |
| "I need more context first"         | Skill check comes BEFORE clarifying questions.         |
| "Let me explore the codebase first" | Skills tell you HOW to explore. Check first.           |
| "I can check git/files quickly"     | Files lack conversation context. Check for skills.     |
| "Let me gather information first"   | Skills tell you HOW to gather information.             |
| "This doesn't need a formal skill"  | If a skill exists, use it.                             |
| "I remember this skill"             | Skills evolve. Read current version.                   |
| "This doesn't count as a task"      | Action = task. Check for skills.                       |
| "The skill is overkill"             | Simple things become complex. Use it.                  |
| "I'll just do this one thing first" | Check BEFORE doing anything.                           |
| "This feels productive"             | Undisciplined action wastes time. Skills prevent this. |
| "I know what that means"            | Knowing the concept ≠ using the skill. Invoke it.      |

## Skill Priority

When multiple skills could apply, use this order:

1. **Process skills first** (brainstorming, debugging) — these determine HOW to approach the task
2. **Implementation skills second** (executing-plans, tdd) — these guide execution

"Let's build X" → brainstorming first, then implementation skills.
"Fix this bug" → debugging first, then domain-specific skills.

## Skill Types

**Rigid** (TDD, debugging): Follow exactly. Don't adapt away discipline.

**Flexible** (patterns): Adapt principles to context.

The skill itself tells you which.

## Available Skills

- nexus:brainstorming — Use when doing creative work — features, components, modifications, or behavior changes
- nexus:writing-plans — Use when you have a spec or requirements for a multi-step task, before touching code
- nexus:executing-plans — Use when you have a written implementation plan to execute
- nexus:test-driven-development — Use when implementing any feature or bugfix, before writing implementation code
- nexus:systematic-debugging — Use when encountering any bug, test failure, or unexpected behavior, before proposing fixes
- nexus:verification-before-completion — Use when about to claim work is complete, before committing or creating PRs
- nexus:requesting-code-review — Use when completing tasks or before merging to verify work meets requirements
- nexus:finishing-a-development-branch — Use when implementation is complete and you need to decide how to integrate the work
- nexus:subagent-driven-development — Use when executing plans with 2+ independent tasks that can run in parallel
- nexus:using-git-worktrees — Use when starting feature work that needs isolation from current workspace
- nexus:writing-skills — Use when creating new skills, editing existing skills, or verifying skills work
- nexus:ui-ux-pro-max — Use when designing UI/UX — layouts, styles, palettes, fonts, charts, components

## User Instructions

Instructions say WHAT, not HOW. "Add X" or "Fix Y" doesn't mean skip workflows.

````javascript
#!/usr/bin/env node

import {
  readFileSync,
  readdirSync,
  existsSync,
  writeFileSync,
  mkdirSync,
  cpSync,
} from "node:fs";
import { join, basename, resolve, dirname } from "node:path";
import { fileURLToPath } from "node:url";

const __filename = fileURLToPath(import.meta.url);
const ROOT = resolve(dirname(__filename), "..");
const SKILLS_DIR = join(ROOT, "skills");
const SPECS_DIR = join(ROOT, "specs", "skills");

// --- Frontmatter parser ---

function parseFrontmatter(content) {
  const lines = content.split("\n");
  if (lines[0].trim() !== "---") return { frontmatter: {}, body: content };
  let end = -1;
  for (let i = 1; i < lines.length; i++) {
    if (lines[i].trim() === "---") {
      end = i;
      break;
    }
  }
  if (end === -1) return { frontmatter: {}, body: content };
  const fm = {};
  for (let i = 1; i < end; i++) {
    const match = lines[i].match(/^(\w+):\s*(.+)$/);
    if (match) fm[match[1]] = match[2].replace(/^["']|["']$/g, "");
  }
  return { frontmatter: fm, body: lines.slice(end + 1).join("\n") };
}

// --- Discover skills ---

function discoverSkills() {
  const skills = [];
  if (!existsSync(SKILLS_DIR)) return skills;
  for (const entry of readdirSync(SKILLS_DIR, { withFileTypes: true })) {
    if (!entry.isDirectory()) continue;
    const skillFile = join(SKILLS_DIR, entry.name, "SKILL.md");
    if (!existsSync(skillFile)) continue;
    const content = readFileSync(skillFile, "utf-8");
    const { frontmatter, body } = parseFrontmatter(content);
    skills.push({
      dir: entry.name,
      path: skillFile,
      frontmatter,
      body,
      content,
    });
  }
  return skills;
}

// --- VALIDATE ---

const PROCESS_SKILLS = new Set([
  "brainstorming",
  "writing-plans",
  "executing-plans",
  "test-driven-development",
  "systematic-debugging",
  "verification-before-completion",
  "requesting-code-review",
  "finishing-a-development-branch",
  "subagent-driven-development",
  "using-git-worktrees",
]);

function validate() {
  const skills = discoverSkills();
  if (skills.length === 0) {
    console.error("No skills found in skills/");
    process.exit(1);
  }

  const skillNames = new Set(skills.map((s) => s.dir));
  let errors = 0;
  let warnings = 0;

  for (const skill of skills) {
    const fm = skill.frontmatter;
    const prefix = `  ${skill.dir}:`;
    const isProcess = PROCESS_SKILLS.has(skill.dir);

    // Frontmatter checks
    if (!fm.name) {
      console.error(`${prefix} ERROR — missing 'name' in frontmatter`);
      errors++;
    }
    if (!fm.description) {
      console.error(`${prefix} ERROR — missing 'description' in frontmatter`);
      errors++;
    }
    if (!fm.whenToUse && skill.dir !== "using-nexus") {
      console.error(`${prefix} ERROR — missing 'whenToUse' in frontmatter`);
      errors++;
    }

    // whenToUse checks
    if (fm.whenToUse) {
      if (!fm.whenToUse.startsWith("Use when")) {
        console.error(`${prefix} ERROR — whenToUse must start with "Use when"`);
        errors++;
      }
      if (fm.whenToUse.length > 250) {
        console.warn(
          `${prefix} WARN — whenToUse exceeds 250 chars (${fm.whenToUse.length})`,
        );
        warnings++;
      }
    }

    // Frontmatter size
    const fmSize =
      (fm.name || "").length +
      (fm.description || "").length +
      (fm.whenToUse || "").length;
    if (fmSize > 1024) {
      console.warn(
        `${prefix} WARN — frontmatter fields exceed 1024 chars (${fmSize})`,
      );
      warnings++;
    }

    // Placeholders
    for (const pattern of ["TBD", "TODO", "..."]) {
      if (skill.body.includes(pattern)) {
        // Exclude '...' in code blocks and tables
        if (
          pattern === "..." &&
          (skill.body.includes("```") || skill.body.includes("|"))
        )
          continue;
        console.error(`${prefix} ERROR — contains placeholder "${pattern}"`);
        errors++;
        break;
      }
    }

    // Enforcement check
    const hasHardGate =
      skill.content.includes("HARD-GATE") ||
      skill.content.includes("HARD_GATE");
    const hasIronLaw = /iron law|NO .+ WITHOUT|NO .+ FIRST/i.test(
      skill.content,
    );
    const hasRedFlags =
      /red flags/i.test(skill.content) && skill.content.includes("| Thought");
    if (!hasHardGate && !hasIronLaw && !hasRedFlags) {
      if (isProcess) {
        console.error(
          `${prefix} ERROR — process skill missing enforcement (HARD-GATE, Iron Law, or Red Flags)`,
        );
        errors++;
      } else if (skill.dir !== "using-nexus") {
        console.warn(`${prefix} WARN — no enforcement mechanism found`);
        warnings++;
      }
    }

    // Chain integrity
    const refs = skill.content.match(/nexus:[\w-]+/g) || [];
    for (const ref of refs) {
      const refName = ref.replace("nexus:", "");
      if (!skillNames.has(refName) && refName !== "using-nexus") {
        console.error(
          `${prefix} ERROR — references nexus:${refName} but skill not found`,
        );
        errors++;
      }
    }
  }

  // Bootstrap sync check
  const bootstrap = skills.find((s) => s.dir === "using-nexus");
  if (bootstrap) {
    for (const skill of skills) {
      if (skill.dir === "using-nexus") continue;
      if (!bootstrap.content.includes(`nexus:${skill.dir}`)) {
        console.warn(
          `  using-nexus: WARN — missing nexus:${skill.dir} in Available Skills list`,
        );
        warnings++;
      }
    }
  }

  console.log(
    `\n${skills.length} skills checked: ${errors} errors, ${warnings} warnings`,
  );
  process.exit(errors > 0 ? 1 : 0);
}

// --- SYNC ---

function sync() {
  if (!existsSync(SPECS_DIR)) {
    console.error(`Specs directory not found: ${SPECS_DIR}`);
    process.exit(1);
  }

  const files = readdirSync(SPECS_DIR).filter((f) => f.endsWith(".md"));
  let created = 0;
  let updated = 0;

  for (const file of files) {
    const content = readFileSync(join(SPECS_DIR, file), "utf-8");
    const name = basename(file, ".md");
    const skillDir = join(SKILLS_DIR, name);
    const skillFile = join(skillDir, "SKILL.md");

    if (!existsSync(skillDir)) mkdirSync(skillDir, { recursive: true });

    const { frontmatter } = parseFrontmatter(content);
    const exists = existsSync(skillFile);

    // Only sync if skill doesn't exist yet (don't overwrite enriched skills)
    if (!exists) {
      writeFileSync(skillFile, content, "utf-8");
      console.log(`  + ${name}/SKILL.md (created)`);
      created++;
    } else {
      console.log(`  ~ ${name}/SKILL.md (exists, skipped)`);
    }

    // Copy auxiliary dirs
    for (const auxDir of ["data", "scripts"]) {
      const srcAux = join(SPECS_DIR, "..", "..", "skills", name, auxDir);
      const destAux = join(skillDir, auxDir);
      if (existsSync(srcAux) && !existsSync(destAux)) {
        cpSync(srcAux, destAux, { recursive: true });
        console.log(`  + ${name}/${auxDir}/ (copied)`);
      }
    }
  }

  console.log(
    `\nSync complete: ${created} created, ${files.length - created} skipped`,
  );
}

// --- GRAPH ---

function graph() {
  const skills = discoverSkills();
  const edges = [];

  for (const skill of skills) {
    // Find REQUIRED SUB-SKILL references
    const subSkillRefs =
      skill.content.match(/REQUIRED SUB-SKILL[^`\n]*nexus:([\w-]+)/g) || [];
    for (const ref of subSkillRefs) {
      const match = ref.match(/nexus:([\w-]+)/);
      if (match)
        edges.push({ from: skill.dir, to: match[1], type: "required" });
    }

    // Find terminal state references
    const terminalRefs =
      skill.content.match(/terminal state[^`\n]*nexus:([\w-]+)/gi) || [];
    for (const ref of terminalRefs) {
      const match = ref.match(/nexus:([\w-]+)/);
      if (match)
        edges.push({ from: skill.dir, to: match[1], type: "terminal" });
    }
  }

  // Generate Mermaid
  let mermaid = "```mermaid\ngraph TD\n";
  for (const skill of skills) {
    if (skill.dir === "using-nexus") continue;
    const id = skill.dir.replace(/-/g, "_");
    mermaid += `    ${id}["${skill.dir}"]\n`;
  }
  mermaid += "\n";
  const seen = new Set();
  for (const edge of edges) {
    const key = `${edge.from}->${edge.to}`;
    if (seen.has(key)) continue;
    seen.add(key);
    const fromId = edge.from.replace(/-/g, "_");
    const toId = edge.to.replace(/-/g, "_");
    const style = edge.type === "required" ? "-->" : "-.->";
    mermaid += `    ${fromId} ${style} ${toId}\n`;
  }
  mermaid += "```\n";

  const docsDir = join(ROOT, "docs");
  if (!existsSync(docsDir)) mkdirSync(docsDir, { recursive: true });

  const output = `# Nexus Skill Graph\n\n> Auto-generated by \`nexus graph\`. Do not edit manually.\n\n${mermaid}`;
  const outPath = join(docsDir, "skill-graph.md");
  writeFileSync(outPath, output, "utf-8");
  console.log(
    `Graph written to docs/skill-graph.md (${edges.length} edges, ${skills.length - 1} skills)`,
  );
}

// --- Main ---

const command = process.argv[2];

switch (command) {
  case "validate":
    validate();
    break;
  case "sync":
    sync();
    break;
  case "graph":
    graph();
    break;
  default:
    console.log(
      "Usage: nexus <command>\n\nCommands:\n  validate  Check all skills for quality standards\n  sync      Generate skills from specs/skills/ source files\n  graph     Generate skill relationship diagram",
    );
    process.exit(command ? 1 : 0);
}
````
